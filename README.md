# 植物大战僵尸 — 网页版

> 基于 HTML5 Canvas + 原生 ES Modules 构建的完整 PvZ 游戏，无需构建工具，无外部依赖。

## 在线游玩 / 本地运行

### 本地运行（推荐）

```bash
git clone https://github.com/195286381/pvz-kimi
cd pvz-kimi
python3 -m http.server 8080
# 打开 http://localhost:8080
```

> 必须通过 HTTP 服务器访问（ES Modules 不支持 `file://` 协议）

## 游戏玩法

1. 点击顶部植物卡片选择植物（需消耗阳光）
2. 点击格子放置已选植物
3. 点击飘落的阳光图标收集阳光
4. 阻止僵尸到达最左侧边界
5. 消灭所有波次的僵尸即通关
6. 共 50 关，5 个场景逐步解锁

## 场景（5个）

| 场景 | 关卡 | 特点 |
|------|------|------|
| 白天草坪 (day) | 1-10 | 标准场景，阳光自然掉落，引入基础僵尸 |
| 夜晚草坪 (night) | 11-20 | 无自然阳光掉落，解锁蘑菇类植物，引入橄榄球/报纸僵尸 |
| 泳池 (pool) | 21-30 | 中间两行为水路，需睡莲才能种植，引入海豚骑手/潜水僵尸 |
| 薄雾泳池 (fog) | 31-40 | 泳池基础上右侧被雾遮挡，僵尸位置难以预判，引入小丑僵尸 |
| 屋顶 (roof) | 41-50 | 射手无法直接攻击，需花盆辅助，引入梯子僵尸 |

## 技术架构

```
src/
├── core/       # 游戏引擎（GameLoop, Renderer, Grid, EventBus, Game）
├── systems/    # 游戏系统（Sun, Wave, Collision, Particles）
├── entities/   # 基类（Plant, Zombie, Projectile）
├── plants/     # 17种植物实现
├── zombies/    # 10种僵尸实现
├── scenes/     # 5个场景 + 工厂函数
├── ui/         # HUD, PlantCard, Screens
└── data/       # 静态数据（plants, zombies, levels）
```

**核心设计原则：**

- 纯 ES Modules，零构建工具，零外部依赖
- 事件总线（EventBus）解耦系统间通信
- 数据驱动：植物/僵尸/关卡完全由数据文件配置
- 对象池（ParticleSystem）避免频繁 GC

## E2E 测试（21项）

```bash
npm install
npx playwright install chromium
python3 -m http.server 8080 &
npx playwright test
```

测试覆盖范围：
- TC-01~08：基础流程（主菜单、关卡选择、游戏运行、暂停）
- TC-09~15：深度游戏逻辑（植物放置、僵尸入场、波次、泳池、浓雾）
- TC-16~17：场景完整性（屋顶场景、favicon）

```bash
npx playwright test --ui
```

## 开发说明

### 架构特点
- **零依赖**：纯 ES Modules，不依赖任何 npm 包（仅 devDependencies 有 Playwright）
- **事件驱动**：所有系统通过 EventBus 松耦合通信
- **数据驱动**：植物/僵尸/关卡完全由 JSON 配置，添加新内容只需改数据
- **对象池**：ParticleSystem 预分配 300 个粒子对象，避免 GC 压力

### 添加新植物
1. `src/data/plants.js` 添加静态数据
2. `src/plants/NewPlant.js` 实现类（继承 Plant）
3. `src/plants/index.js` 注册到 PLANT_REGISTRY
4. `src/ui/HUD.js` SCENE_PLANTS 中添加到对应场景
