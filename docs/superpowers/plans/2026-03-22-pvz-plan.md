# 植物大战僵尸网页游戏 — 实现计划

**日期：** 2026-03-22
**依据：** `docs/superpowers/specs/2026-03-22-pvz-design.md`
**技术栈：** HTML5 Canvas + 原生 ES Modules（无构建工具）

---

## 实现策略

采用**由核心向外扩展**的分阶段策略：先打通游戏循环和最小可玩原型，再逐步添加植物/僵尸/场景，最后完善UI和关卡内容。每个阶段结束后应有可运行的游戏状态。

---

## 阶段一：项目骨架 + 游戏循环

**目标：** 浏览器打开 `index.html` 能看到 Canvas 画布并运行游戏循环。

### 任务

- [ ] **1.1** 创建 `index.html`：Canvas 元素（800×600）、ES Module 入口
- [ ] **1.2** `src/core/EventBus.js`：简单发布/订阅，`on/off/emit`
- [ ] **1.3** `src/core/GameLoop.js`：`requestAnimationFrame` 循环，计算 `deltaTime`（上限50ms），暴露 `start/stop`
- [ ] **1.4** `src/core/Renderer.js`：封装 Canvas context，提供 `clear()`、`drawText()`、`drawRect()` 等基础方法
- [ ] **1.5** `src/core/Game.js`：主类，持有所有系统引用，`update(dt)` / `render()` 空框架
- [ ] **1.6** 联调：`index.html` 初始化 `Game`，`GameLoop` 驱动，控制台输出 fps

**验收：** 控制台稳定输出约 60fps，Canvas 黑色背景正常显示。

## 阶段四：僵尸基础系统 + 碰撞检测

**目标：** 3种基础僵尸（普通/路锥/铁桶）能入场、移动、攻击植物，子弹能打伤/消灭僵尸。

### 任务

- [ ] **4.1** `src/entities/Zombie.js`：僵尸基类（hp、armorHp、speed、damage、attackInterval、状态机：移动/攻击/死亡）
- [ ] **4.2** `src/systems/CollisionSystem.js`：
  - 子弹 × 僵尸：同行 AABB 检测，命中扣血，子弹消除
  - 僵尸 × 植物：僵尸到达植物列时转入攻击状态
  - 僵尸到达左侧边界：触发游戏失败事件
- [ ] **4.3** `src/zombies/BasicZombie.js`：普通僵尸（HP=270，慢速）
- [ ] **4.4** `src/zombies/ConeZombie.js`：路锥僵尸（护甲370，路锥被打掉后变普通）
- [ ] **4.5** `src/zombies/BucketZombie.js`：铁桶僵尸（护甲1100）
- [ ] **4.6** `src/systems/WaveSystem.js`：按关卡配置生成波次，支持普通波/旗手波，波次进度条
- [ ] **4.7** `src/zombies/FlagZombie.js`：旗手僵尸（触发大波次标记）
- [ ] **4.8** 胜利条件：所有波次僵尸清除后延迟2s触发胜利

**验收：** 完整一局白天草坪可玩通：僵尸入场→豌豆射手打僵尸→僵尸吃植物→通关/失败逻辑正常。

## 风险与注意事项

| 风险 | 应对 |
|------|------|
| 梯子僵尸状态复杂 | 坚果墙格子维护 `hasLadder` 布尔值，与高坚果分支处理 |
| 磁力菇需扫描全行 | 每3s触发一次，不在每帧检测，性能可控 |
| 火炬树桩与冰豌豆交互 | 子弹生成时检查同行是否有火炬树桩；冰豌豆遇火炬变普通豌豆 |
| 屋顶僵尸入场方式不同 | RoofScene 重写僵尸生成位置（从顶部弧线落下） |
| 泳池渲染分层 | 渲染顺序：睡莲层 → 陆地植物层 → 子弹层 → 僵尸层 |
