# 植物大战僵尸网页游戏 — 设计文档

**日期：** 2026-03-22
**技术栈：** HTML5 Canvas + 原生 ES Modules（无构建工具）
**目标：** 完整还原经典 PvZ 玩法，卡通/Q版风格，5大地图场景

---

## 1. 产品概述

### 目标
构建一个基于浏览器的植物大战僵尸完整版游戏，使用 HTML5 Canvas 渲染，无需任何外部图片资源（全用 Emoji + Canvas 绘图实现卡通/Q版画风），无构建工具，直接用 ES Modules 模块化组织代码。

### 核心体验
- 玩家在格子上放置植物抵御从右侧入场的僵尸
- 向日葵产出阳光，阳光用于购买植物
- 清除所有波次僵尸即通关，任意一只僵尸到达左侧边界则游戏失败
- 共50关，跨越5个场景地图

### 不包含
- 存档/进度系统（每次游玩独立）
- 联机功能
- 商店/内购系统

---

## 2. 视觉风格

**卡通/Q版风格**，接近原版 PvZ 美术方向：
- 色彩鲜艳，圆润可爱
- 植物和僵尸使用 Emoji + Canvas 自绘图形表现
- 背景使用 Canvas 渐变和几何图形营造各场景氛围
- 子弹、爆炸、阳光等使用粒子系统实现动画效果

## 3. 游戏网格

| 场景 | 行数 | 列数 | 特殊格子 |
|------|------|------|----------|
| 白天草坪 | 5 | 9 | 无 |
| 夜晚草坪 | 5 | 9 | 无 |
| 泳池 | 6 | 9 | 第2、3行（0-indexed）为水路，其余4行为陆地 |
| 浓雾 | 5 | 9 | 右侧5列（col 4-8）视野遮罩（视觉遮盖，植物仍可攻击进入射程的僵尸） |
| 屋顶 | 5 | 9 | 斜面，需先放花盆再种植物 |

网格坐标系：`(row, col)`，`row` 0 起始从上到下，`col` 0 起始从左到右。

**泳池场景说明：**
- row 0、1、4、5 为陆地格，可直接种植任意陆地植物
- row 2、3 为水路格，只能先种睡莲，再在睡莲上叠放其他植物（睡莲本身不攻击）
- 渲染顺序：先渲染睡莲，再渲染上层植物（上层植物覆盖睡莲）

## 6. 场景系统（5个）

### 白天草坪（关卡 1-10）
- 标准场景，阳光每10s（±2s随机浮动）从天空随机列掉落
- 5行×9列格子
- 僵尸波次：第1-5关2波，第6-10关4波

### 夜晚草坪（关卡 11-20）
- 无天空阳光掉落，需靠向日葵/阳光菇产出
- **小喷菇和毁灭菇**在夜晚免费（费用=0）；阳光菇仍需25阳光（但只能在夜晚场景种植）
- 屏幕变暗，背景改为夜空

### 泳池（关卡 21-30）
- 6行格子，row 2、3（0-indexed，即视觉上第3、4行）为蓝色水路
- 水路格子只能种睡莲（睡莲上再种其他植物）
- 出现海豚骑手、潜水僵尸

### 浓雾（关卡 31-40）
- 右侧4列被雾遮盖（植物无法看到此区域内的僵尸）
- 浓雾为**纯视觉遮罩**：植物仍可正常攻击进入其射程的僵尸，但玩家无法预览遮罩内的僵尸位置
- 无消雾道具（简化设计），遮罩贯穿全关
- 出现小丑僵尸

### 屋顶（关卡 41-50）
- 斜面背景，普通植物需花盆才能种植（额外消耗25阳光）
- 僵尸从屋顶右侧投石车空降
- 出现小丑僵尸、橄榄球僵尸

## 7. 波次系统

每关卡包含若干波次：
- **普通波次**：生成若干随机僵尸
- **大波次**（旗手波）：大量僵尸同时入场，旗手僵尸先行
- 波次间隔：上一波次僵尸清除后延迟5s，或超时20s强制下一波
- 波次进度条实时显示

关卡配置示例（`data/levels.js`）：
```js
{
  id: 1,
  scene: 'day',
  waves: [
    { zombies: ['basic', 'basic', 'basic'], delay: 0 },
    { zombies: ['basic', 'cone', 'basic', 'basic'], delay: 0, isFlagWave: true }
  ]
}
```

## 8. 阳光系统

- **天空阳光**：白天场景每10s（±2s随机浮动）在随机列从顶部掉落，玩家点击收集+25
- **向日葵产阳光**：每24s生成一个阳光，自动漂浮等待点击
- **阳光菇**（夜晚）：小型25/中型50/大型75，生长需时
- **阳光上限**：9990；达到上限时天空阳光停止掉落，向日葵/阳光菇产出的阳光仍显示但不计入（超出丢弃）；HUD 数字显示金色闪烁提示满载
- **点击收集动画**：阳光飘向HUD阳光计数器

## 9. UI 系统

### HUD（游戏中界面）
- 顶部：阳光数量（黄色数字）+ 植物卡片栏
- 植物卡片：Emoji图标 + 费用 + 冷却进度条（灰色遮罩倒计时）
- 右上：波次进度条 + 当前波次文字
- 右上角：暂停按钮

### 植物选择交互
1. 点击植物卡片 → 卡片高亮，鼠标跟随植物预览
2. 点击有效格子 → 放置植物，扣除阳光，卡片进入冷却
3. 点击无效区域/右键 → 取消选择

### 其他界面
- **主菜单**：标题 + 开始游戏 + 关卡选择
- **关卡选择**：5个场景分组，各10关
- **暂停菜单**：继续/重试/返回主菜单
- **胜利界面**：动画 + 进入下一关
- **失败界面**：重试/返回主菜单

### 文件结构

### 游戏循环
```
requestAnimationFrame → GameLoop.tick(timestamp)
  → deltaTime = min(timestamp - lastTime, 50ms)  // 限制最大dt防止卡顿跳变
  → Game.update(dt)
      ├── SunSystem.update(dt)
      ├── WaveSystem.update(dt)
      ├── plants.forEach(p => p.update(dt))
      ├── zombies.forEach(z => z.update(dt))
      ├── projectiles.forEach(p => p.update(dt))
      ├── CollisionSystem.check(projectiles, zombies, plants)
      └── ParticleSystem.update(dt)
  → Game.render()
      ├── Scene.render(ctx)       // 背景
      ├── plants.forEach render   // 植物层
      ├── projectiles.forEach render
      ├── zombies.forEach render  // 僵尸层
      ├── ParticleSystem.render(ctx)
      └── HUD.render(ctx)        // UI最顶层
```

### 碰撞检测
- **子弹 × 僵尸**：同行 AABB 检测，子弹 x 坐标与僵尸 x 重叠
- **僵尸 × 植物**：僵尸到达植物所在列格子时触发攻击
- **阳光收集**：鼠标点击坐标与阳光圆形区域检测

### 性能考量
- 实体数量上限：植物45个（5×9）、子弹约100个、僵尸约50个
- 每帧统一在Canvas上重绘，无DOM操作
- 粒子系统对象池复用，避免频繁GC

## 11. 数据驱动设计

植物和僵尸均使用静态数据配置 + 继承类实现，便于扩展：

```js
// data/plants.js 示例
export const PLANTS = {
  sunflower: { cost: 50, cooldown: 7, hp: 300, sunInterval: 24, sunAmount: 25 },
  peashooter: { cost: 100, cooldown: 7, hp: 300, attackInterval: 1.5, damage: 20 },
  // ...
}
```

关卡配置完全数据驱动，新增关卡只需在 `levels.js` 添加条目。
